# 시장 분석 및 경쟁 포지셔닝

## 시장 전제

- **핵심 전제**: 1인 개발자 + AI 에이전트 팀 = 5인 팀 이상의 결과물. 이 생산성 격차를 상품으로 판매한다.
- **타깃 구간**: 중단가·빠른 납기를 원하지만 일반 프리랜서의 불안정성과 대형 에이전시의 오버헤드/속도를 피하고 싶은 조직.

## 포지셔닝 (요약)

| 구분 | 일반 프리랜서 | 에이전시 | 본 전략 (1인 + 에이전트) |
|------|----------------|----------|---------------------------|
| 단가 | 낮음 | 높음, 느림 | 중~고, 빠름 |
| 품질 | 불안정 | 팀 보정 | AI 에이전트 팀으로 표준화 |
| 구조 | 1인 한계 | 커뮤니케이션 복잡 | 오케스트레이터 중심, 병렬 산출 |

## 핵심 자산 → 시장 메시지

| 자산 | 설명 | 수익화 강도 |
|------|------|-------------|
| 방산·항공 CBT 개발 | 인터랙티브 교육 시스템(오프라인·보안 요건 대응 경험) | 높음 |
| Electron + React | 데스크톱·풀스택 실행력 | 높음 |
| Harness Engineering | 에이전트 팀 설계·운용 | 높음 |
| 바이브코딩 방법론 | 1인 → 팀 규모 생산성 | 중~고 |
| LMS/LCMS | 교육 플랫폼 설계 | 중~고 |

## 아이디어별 페르소나

### 아이디어 A: CBT 자동생성 SaaS

```yaml
primary:
  - 방산 업체 교육팀 (예: KAI, 한화, LIG넥스원)
  - 항공사 운항훈련팀
  - 제조업 안전교육 담당자
secondary:
  - 정부/공공기관 사이버교육 담당
  - 군 교육사령부
```

**가설**: 스크립트 업로드 → AI 파이프라인 → 오프라인 실행 가능 패키지(SCORM/Electron)는 보안·망분리 환경 요구와 맞닿아 있다.

### 아이디어 B: 바이브코딩 에이전시

- **Primary**: 스타트업 창업자, 사내 혁신 담당, MVP가 급한 IT 리더.
- **메시지**: 2주 MVP, Harness 기반 품질, Electron·Next 전문.

### 아이디어 C: 교육·커뮤니티

- **Primary**: AI 코딩 도입을 원하는 1인 개발자·팀 리드.
- **Secondary**: 방산/교육 테크에 관심 있는 실무자.

## 경쟁·대체재 (가정)

- **수작업 CBT 제작사/내부 제작**: 단가·기간 높음; 자동화가 차면 “첫 초안 속도”에서 차별.
- **일반 LLM 챗**: 산출물 일관성·SCORM·오프라인 패키징·감사 추적 부재 → **엔터프라이즈 워크플로**가 차별.
- **국내외 저작도구/LMS**: 연동·온프레미스·방산 요건 맞춤이 병목일 수 있음.

## 디스커버리 인터뷰 가이드 (요약)

1. 현재 교육 콘텐츠 제작 주기와 담당 조직은?
2. LMS·망분리·오프라인 실행 요구는?
3. 기존 CBT/SCORM 운영 시 불만(비용, 수정, 배포)?
4. “지금 당장 작은 예산으로 시험” 가능 여부와 결정권.

상세 검증 체크리스트는 [DESIGN.md](../DESIGN.md#phase-6-검증-체크리스트)를 참조한다.
